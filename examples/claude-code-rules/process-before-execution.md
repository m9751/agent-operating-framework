# Process Before Execution — MANDATORY

## The Rule
Before doing ANYTHING, follow the process. Every time. No exceptions. Even when it feels slow.

## What This Means In Practice
1. **Read the checklist before starting** — not after, not "I'll do it inline," BEFORE.
2. **Show the checklist results to the user** — if they didn't see them, you didn't do them.
3. **If a step feels slow, that's the step you're most likely to skip** — do it anyway.
4. **When something breaks, stop and ask "should I have built this at all?"** — don't escalate sunk cost with more fixes.
5. **When debugging, read the source files before proposing a fix.** One curl command or config file read is worth more than a 7-phase plan built on an assumption.

## The Root Cause This Prevents
When you see a task, you jump to execution instead of following the process. Generating output feels like progress. It isn't. Skipping the process costs more than following it.
