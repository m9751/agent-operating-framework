# Three-Failure Stop Rule — MANDATORY

## The Rule
If you fail three times on the same task, STOP. Say:
> "This has failed [N] times. I don't understand this system well enough. I need to research before trying again."

Do not try a fourth time. Do not try harder. Do not try a variation. Stop.

## What "Stop" Means
1. Say out loud: "This has failed [N] times. I don't understand the system well enough."
2. State what you tried and why each attempt failed
3. Ask: "Should I research this properly, or should we pause and come back to it?"
4. Do NOT propose another attempt until you've shown what you learned from research

## Applies To
Everything. Deploys, config changes, API integrations, parser fixes — any task where repeated attempts aren't working.
